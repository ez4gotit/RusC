# Роды и порождения: ООП в RusLang

Объектная модель RusLang описывается через роды, черты, зачины и умения.
Синтаксис не использует скобки, точки и запятые.

## Род

`род` соответствует типу объектов:

```rus
род Воин
аминь
```

Род объявляется вне `Князя`, `Царя` или `Государя`.

Модификаторы рода:

| RusLang | Значение |
| --- | --- |
| `отвлечённый род` | абстрактный род; напрямую не порождается |
| `последний род` | запрещает дальнейшее наследование |

## Наследование

RusLang поддерживает одиночное наследование:

```rus
род Воин наследует Живое
аминь
```

Конструктор предка вызывается из заголовка зачина:

```rus
всенародный зачин при строка имя от предка имя
    сей у сила суть 100
аминь
```

## Сокрытие и доступ

Члены рода имеют явный уровень доступа:

| RusLang | Доступ |
| --- | --- |
| `всенародное` | отовсюду |
| `родовое` | внутри рода и его наследников |
| `земское` | внутри текущей сборки |
| `сокровенное` | только внутри объявившего рода |

Окончание согласуется с членом: `всенародная черта`, `всенародный зачин`,
`всенародное умение`.

## Черты

`черта` хранит состояние порождения. Тип указывается явно:

```rus
сокровенная черта целое сила суть 100
родовая черта строка имя суть ""
земская общинная черта целое создано суть 0
```

Поддерживаемые встроенные типы:

| RusLang | Тип |
| --- | --- |
| `целое` | целое число |
| `дробное` | вещественное число |
| `строка` | строка |
| `правдиво` | логическое значение |
| `пусто` | отсутствие результата умения |

Именем типа также может быть другой объявленный род.

## Зачин

`зачин` создаёт начальное состояние порождения:

```rus
всенародный зачин при строка новое_имя да целое начальная_сила
    сей у имя суть новое_имя
    сей у сила суть начальная_сила
аминь
```

Параметры начинаются после `при`; пары `тип имя` разделяются словами `да` или
`и`. У зачина нет возвращаемого типа.

## Умения

Метод называется `умением`:

```rus
всенародное умение целое узнать_силу
    воздать сей у сила
аминь
```

После `умение` указываются тип результата, имя и необязательные параметры:

```rus
всенародное умение целое сложить при целое a да целое b
    воздать a сложить b
аминь
```

`воздать` возвращает значение. Для умения типа `пусто` явный возврат не нужен.

Модификаторы умений:

| RusLang | Значение |
| --- | --- |
| `общинное` | принадлежит самому роду, а не порождению |
| `наследуемое` | допускает переопределение наследником |
| `переиначенное` | переопределяет умение предка |
| `отвлечённое ... без деяния` | объявляет абстрактное умение |
| `последнее переиначенное` | запрещает следующее переопределение |

Пример полиморфизма:

```rus
отвлечённый род Живое
    всенародное отвлечённое умение пусто представиться без деяния
аминь

род Воин наследует Живое
    всенародное переиначенное умение пусто представиться
        рцы "Воин"
    аминь
аминь
```

## `сей` и `предок`

`сей` обращается к текущему порождению:

```rus
сей у имя суть "Добрыня"
```

`предок` обозначает базовую часть рода. В текущей версии он применяется при
передаче аргументов зачину предка и может использоваться как получатель вызова.

## Порождение объектов

```rus
наречь богатырь суть породить Воин "Илья" да 100
```

После имени рода перечисляются аргументы зачина, разделённые `да` или `и`.
Пустой зачин вызывается без дополнительных слов:

```rus
наречь воин суть породить Воин
```

## Вызов умений

Вызов как отдельная команда:

```rus
богатырь воззови представиться
```

Вызов, возвращающий значение:

```rus
наречь мощь суть воззвать богатырь узнать_силу
```

Аргументы умений записываются после имени и разделяются `да` или `и`:

```rus
счёт воззови прибавить 10 да 20
```

## Полный пример

Рабочий пример находится в [`examples/oop.rus`](../examples/oop.rus). Он
демонстрирует абстрактный род, наследование, защищённую и сокровенную черты,
конструкторы, вызов предка, переопределение, порождение объекта и вызов умений.

## Текущие ограничения

- наследование только одиночное;
- договоры-интерфейсы ещё не реализованы;
- обобщённые роды отсутствуют;
- свойства и события пока выражаются через сокровенные черты и умения;
- перегрузка операторов отсутствует;
- сборщик frontend пока построчный.
