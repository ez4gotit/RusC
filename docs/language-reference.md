# RusLang: полный справочник языка

RusLang — русскоязычный системный язык, развиваемый как самостоятельная
альтернатива C++. Его синтаксис строится не как перевод английских ключевых
слов, а вокруг образов древней Руси:

- внешние дружины **призываются**, когда они действительно нужны;
- исполняемой частью программы правит **Князь**, **Царь** или **Государь**;
- стандартные возможности доступны сразу, без пустых обрядовых импортов;
- служебные запятые, скобки и символьные операторы не используются.

Документ описывает весь синтаксис, который поддерживает текущий frontend.
Исходные файлы имеют расширение `.rus`, кодировку UTF-8 и обрабатываются
построчно. Регистр ключевых слов не имеет значения.

## Полная минимальная программа

```rus
Князь
    рцы "Привет, мир!"
аминь
```

`Князь`, `Царь` и `Государь` — равнозначные имена единственной точки входа.
Исполняемые команды находятся внутри неё до слова `конец`, `аминь` или
`совершено`.

Отступы необязательны, но рекомендуются. Точка с запятой не ставится.

## Запрет служебной пунктуации

В коде RusLang вне строк запрещены:

```text
(  )  [  ]  {  }  ,  =  !  <  >  +  -  *  /  %
```

Вместо них используются слова:

| Классическая запись | RusLang |
| --- | --- |
| `1, 2, 3` | `1 и 2 и 3` |
| `[1, 2, 3]` | `ряд 1 и 2 и 3` |
| `длина(числа)` | `длина числа` |
| `соединить(" ", числа)` | `соединить " " и числа` |
| `числа[i]` | `числа по i` |
| `значение = 5` | `значение есть 5` или `значение это 5` |
| `a == b` | `a точно b` или `a бля буду b` |
| `a != b` | `a не есть b` или `a это не b` |

Запятая внутри строкового значения является обычным текстом и разрешена:

```rus
печать "Привет, мир!"
```

Группирующих скобок нет. Для изменения порядка вычислений результат выражения
следует сохранить в промежуточную переменную.

## Призыв внешних дружин

Обычной программе призывы не нужны: вывод, ряды, условия и циклы входят в основу
языка. `призвать` зарезервировано для будущих внешних дружин и ставится перед
точкой входа:

```rus
призвать Берегиню
```

Это аналог подключения зависимости, но часть собственной модели RusLang, а не
калька с `include`, `using` или `import`.

Компилятор принимает имя модуля как идентификатор. Разрешение внешних модулей
и пользовательских дружин ещё не реализовано; поэтому сейчас `призвать` только
фиксирует намерение и не открывает дополнительные имена.

## Точка входа

В одном файле допускается одна точка входа:

```rus
Государь
    вещай "Начало"
совершено
```

Команды вне точки входа запрещены. При обычном завершении программа возвращает
процессу код `0`.

## Комментарии

Строка, первый непробельный символ которой — `#`, является комментарием:

```rus
# Это комментарий
печать "Эта строка выполняется"
```

Комментарий занимает всю строку. Комментарии после команды пока не
поддерживаются.

## Стили и синонимы

RusLang допускает современную русскую и древнерусскую стилизованную запись.
Формы внутри одной строки таблицы равнозначны и могут свободно сочетаться.

| Назначение | Основная форма | Синонимы |
| --- | --- | --- |
| точка входа | `Князь` | `Царь`, `Государь` |
| завершение блока | `аминь` | `конец`, `совершено` |
| объявление | `наречь` | `пусть`, `да будет` |
| присваивание | `суть` | `есть`, `это`, `бысть` |
| условие | `аще` | `если` |
| альтернативная ветвь | `инако` | `иначе` |
| цикл с условием | `доколе` | `пока` |
| счётный цикл | `ступай` | `для` |
| вывод | `рцы` | `молви`, `вещай`, `печать` |
| вывод ошибки | `возопи` | `ошибка` |
| завершение программы | `изыди` | `выход` |
| прерывание цикла | `пресеки` | `прервать` |
| следующая итерация | `далее` | `продолжить` |
| последовательность | `строй` | `ряд`, `полк` |
| размер | `мера` | `длина` |
| соединение | `сочетать` | `соединить`, `совокупить` |
| доступ по индексу | `на месте` | `по` |
| истина | `правда` | `истина`, `истинно` |
| ложь | `кривда` | `ложь`, `ложно` |
| логическое «и» | `да` | `и` |
| логическое «или» | `али` | `или` |
| сложение | `сложить` | `плюс` |
| вычитание | `отъять` | `отнять`, `минус` |
| умножение | `множить` | `умножить` |
| деление | `делить` | `разделить` |
| равенство | `воистину`, `суть` | `точно`, `бля буду` |
| неравенство | `не суть`, `суть не` | `не есть`, `это не` |
| больше | `паче` | `больше` |
| меньше | `менее` | `меньше` |
| увеличение на один | `возрасти` | `увеличить`, `плюс плюс` |
| уменьшение на один | `умалить` | `уменьшить`, `минус минус` |

Это художественная стилизация под старославянскую и древнерусскую лексику, а
не попытка выдать синтаксис за историческую реконструкцию разговорной речи.

## Значения

Поддерживаются:

- целые числа: `42`, `минус 7`;
- вещественные числа: `3.14`;
- строки: `"текст"`;
- логические значения: `истина`, `ложь`;
- ряды: `ряд 5 и 2 и 9 и 1`.

Строковые литералы поддерживают escape-последовательности `\"`, `\\`, `\n`,
`\r`, `\t`.

## Переменные

Переменная объявляется словом `пусть`; её тип выводится из значения:

```rus
пусть счётчик есть 0
пусть имя это "Анна"
пусть готово есть истина
пусть числа это ряд 4 и 2 и 8
```

Повторное присваивание:

```rus
счётчик есть счётчик плюс 1
имя это "Мария"
```

`есть` и `это` равнозначны. Они используются для объявления и обычного
присваивания, но не для сравнения.

Тип переменной после объявления не меняется. Имена могут содержать русские и
латинские буквы, цифры и `_`, но не могут начинаться с цифры.

## Ряды

Ряды входят в основу языка и не требуют призыва:

```rus
Князь
    наречь числа суть строй 4 да 2 да 8
аминь
```

Элементы перечисляются словом `и`, без запятых и скобок. Все элементы должны
иметь совместимый тип. Размер ряда после создания не меняется.

Доступ к элементу:

```rus
пусть первое есть числа по 0
числа по 1 это 10
```

Индекс начинается с нуля. После `по` указывается число или имя переменной.
Сложный индекс сначала вычисляется отдельно:

```rus
пусть следующий есть индекс плюс 1
пусть значение это числа по следующий
```

Выход за границы ряда завершает программу ошибкой среды выполнения.

## Выражения

Арифметические операции записываются словами:

| Операция | RusLang |
| --- | --- |
| сложение | `плюс` |
| вычитание и отрицание | `минус` |
| умножение | `умножить` |
| деление | `разделить` |
| остаток от деления | `остаток` |

Сравнения:

| Смысл | Основная форма | Допустимая форма |
| --- | --- | --- |
| равно | `точно` | `бля буду` |
| не равно | `не есть` | `это не` |
| больше | `больше` | — |
| меньше | `меньше` | — |
| больше или равно | `не меньше` | — |
| меньше или равно | `не больше` | — |

Логические операции:

```text
не  и  или
```

Приоритет от высокого к низкому:

1. `не` и `минус` перед одним значением;
2. `умножить`, `разделить`, `остаток`;
3. `плюс`, `минус`;
4. сравнения;
5. `и`;
6. `или`.

Группирующих скобок нет. Неоднозначное вычисление разбивается на шаги:

```rus
пусть сумма есть a плюс b
пусть результат это сумма умножить c
```

### Составные присваивания

Основные формы:

```rus
счётчик плюс есть 5
счётчик минус есть 2
счётчик умножить есть 3
счётчик разделить есть 2
счётчик остаток есть 4
```

Разрешены более разговорные варианты:

```rus
счётчик прибавить 5
счётчик убавить 2
счётчик умножить на 3
счётчик разделить на 2
счётчик остаток от 4
```

Увеличение и уменьшение на единицу:

```rus
счётчик плюс плюс
счётчик минус минус
счётчик увеличить
счётчик уменьшить
```

Каждая пара форм равнозначна.

## Вывод

Стандартный поток:

```rus
печать "Привет"
печать счётчик
```

Поток ошибок:

```rus
ошибка "Файл не найден"
ошибка код
```

Обе команды доступны без призыва и добавляют перевод строки.

Для совместимости простой текст можно выводить без кавычек:

```rus
печать Привет мир
```

Вычисляемые строки лучше заключать в кавычки или получать встроенной командой,
чтобы запись оставалась однозначной.

## Встроенные команды Сварога

### `длина`

Возвращает размер ряда:

```rus
пусть размер есть длина числа
```

В выражении команда записывается без скобок:

```rus
пусть последний есть длина числа минус 1
```

### `соединить`

Создаёт строку из элементов ряда:

```rus
печать соединить " " и числа
```

Первый аргумент — строка-разделитель, второй — ряд. Аргументы разделяются
словом `и`.

## Условия

```rus
если температура меньше 0
    печать "Мороз"
иначе
    печать "Тепло"
конец
```

Ветка `иначе` необязательна. Блок закрывается словом `конец`.

## Цикл `пока`

```rus
пусть i есть 0
пока i меньше 10
    печать i
    i увеличить
конец
```

## Цикл `для`

```rus
для i от 0 до 10
    печать i
конец
```

Начальная граница включается, конечная — нет. Пример обходит значения от `0`
до `9`. Переменная цикла создаётся автоматически и увеличивается на единицу.

Граница может быть выражением:

```rus
для i от 0 до длина числа
    печать числа по i
конец
```

## Управление циклом

Внутри `пока` и `для` доступны:

```rus
продолжить
прервать
```

`продолжить` начинает следующую итерацию. `прервать` завершает ближайший цикл.

## Завершение программы

```rus
выход 0
```

Аргумент — целочисленное выражение, которое становится кодом завершения
процесса.

## Вложенность

Каждая точка входа, условие и цикл требуют собственного завершителя:

```rus
Князь
    ступай место от 0 до мера числа
        аще числа на месте место паче 0
            рцы числа на месте место
        аминь
    аминь
аминь
```

## Полная грамматика команд

```text
программа       := { призыв } точка_входа { команда } завершитель
призыв          := "призвать" имя
комментарий     := "#" текст
точка_входа     := "Князь" | "Царь" | "Государь"
завершитель     := "аминь" | "конец" | "совершено"

команда         := комментарий
                 | объявление имя связка выражение
                 | цель связка выражение
                 | цель составное_присваивание выражение
                 | цель изменение_на_единицу
                 | вывод значение_вывода
                 | вывод_ошибки значение_вывода
                 | завершение выражение
                 | условие выражение { команда }
                   [ альтернатива { команда } ] завершитель
                 | цикл_условие выражение { команда } завершитель
                 | цикл_счётный имя "от" выражение "до" выражение
                   { команда } завершитель
                 | продолжение
                 | прерывание

объявление      := "наречь" | "пусть" | "да будет"
вывод           := "рцы" | "молви" | "вещай" | "печать"
вывод_ошибки    := "возопи" | "ошибка"
завершение      := "изыди" | "выход"
условие         := "аще" | "если"
альтернатива    := "инако" | "иначе"
цикл_условие    := "доколе" | "пока"
цикл_счётный    := "ступай" | "для"
продолжение     := "далее" | "продолжить"
прерывание      := "пресеки" | "прервать"
цель            := имя ( "по" | "на месте" ) индекс | имя
связка          := "суть" | "бысть" | "есть" | "это"
составное_присваивание
                := "плюс есть" | "минус есть"
                 | "умножить есть" | "разделить есть" | "остаток есть"
                 | "прибавить" | "убавить"
                 | "умножить на" | "разделить на" | "остаток от"
изменение_на_единицу
                := "плюс плюс" | "минус минус"
                 | "увеличить" | "уменьшить"
ряд              := ( "строй" | "ряд" | "полк" ) выражение
                    { ( "да" | "и" ) выражение }
индекс           := целое | имя
значение_вывода := выражение | текст_без_кавычек
```

```rus
            наречь следующий суть индекс сложить 1

                наречь временное бысть числа на месте индекс

    рцы сочетать " " да числа

```

## Отладочный режим C#

Директива `#csharp` в первой строке передаёт оставшийся файл напрямую Roslyn.
На такой файл правила синтаксиса RusLang не распространяются. Это внутренний
режим проверки backend, а не часть языка и не рекомендуемый способ написания
программ.

## Текущие ограничения

Цель проекта — системный язык уровня и выше C++, но текущая реализация остаётся
ранним frontend-прототипом. Пока отсутствуют:

- пользовательские функции, структуры и типы;
- указатели, управление временем жизни и модель владения;
- обобщения и метапрограммирование;
- разрешение внешних призываемых модулей;
- ввод с консоли;
- динамические ряды;
- многофайловая компиляция.

Документация перечисляет только уже работающий синтаксис. Эти возможности
должны проектироваться отдельно, без скрытого проброса синтаксиса C++ или C#.
